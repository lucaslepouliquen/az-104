### 5.4 Azure Network Watcher
- **Fonction** : Monitoring et diagnostic réseau
- **Capacités** : 
  - **IP Flow Verify** : Permet de simuler un flux réseau et de vérifier si celui-ci est autorisé (Allow) ou bloqué (Deny) en fonction des règles NSG (Network Security Group) et des règles d'administration Azure Virtual Network Manager. <br>
                         Données analysées : Adresse IP source, Adresse IP destination, Port source, Port destination, Protocole (TCP/UDP), Direction du trafic (Inbound/Outbound). <br>
                         Résultat fourni : Autorisé ou refusé, Règle NSG concernée, Nom du NSG impactant la communication. 
  - **NSG Flow Logs** : Collecte les métadonnées des flux IP traversant un Network Security Group afin d'analyser les communications réseau réelles. Les logs sont stockés dans un compte Azure Storage et peuvent être exploités par Traffic Analytics.
                        Informations collectées : IP source/destination, Ports source/destination, Protocole, Décision NSG (Allow/Deny), Volume de trafic, Horodatage du flux.
                        Cas d'usage : Audit réseau. Analyse des flux applicatifs. Détection d'anomalies. Investigations de sécurité. Optimisation des règles NSG.
  - **Packet Capture** : Capture de paquets réseau pour troubleshooting
  - **Connection Monitor** : Monitoring continu de la connectivité et latence réseau (Azure + hybrid)
- **Usage** : Troubleshooting connectivité et diagnostic réseau
- **Scope** : Ressources réseau uniquement (VNets, NSGs, VMs)
